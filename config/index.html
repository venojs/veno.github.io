配置文件 #

和vite一样, 配置文件不是必须的, 但正式的项目往往会有比较多的自定义配置

配置文件解析 #

Veno启动时会读取veno.config.js或者veno.config.ts里面的内容

export default {
    // ...
}

可以使用defineConfig来增加智能提示

import { defineConfig } from '@veno/kit';

export default defineConfig({
    // ...
})

配置文件中的环境变量 #

Veno启动时会读取.env文件挂载到process.env(就像Vue Cli做的那样)

如果process.env上MODE COMMAND NODE_ENV这三个的值都不存在, 则会为他们赋值, 如果他们已存在, 则可以加VENO_前缀代替

比如有.env.local文件如下, 并运行veno dev

FOO=bar
VITE_SOME_KEY=123

// veno.config.js
console.log(process.env.FOO) // bar
console.log(process.env.VITE_SOME_KEY) // 123
console.log(process.env.MODE) // development
console.log(process.env.VENO_MODE) // development
console.log(process.env.COMMAND) // serve
console.log(process.env.VENO_COMMAND) // serve
console.log(process.env.NODE_ENV) // development
console.log(process.env.VENO_NODE_ENV) // development

export default {
    // ...
}

提示

  • process.env上的变量仅限在配置文件中使用
  • 只有已VITE_开头变量才能在客户端代码中使用, 使用方式见vite官网

配置选项 #

vite #

vite选项的配置与vite.config的配置一致

提示

  • vite选项里配置插件会与Veno内置的插件合并

router #

内置的router插件

  • 类型： boolean | RouterPluginOptions
  • 默认： true
export interface RouterPluginOptions {
    mpa?: boolean; // 是否是mpa模式
    base?: string; // 路由前缀 默认值 "/"
    pagesDir?: string; // 路由文件夹 默认值 src/pages
    filename?: string; // 扫描指定文件名来生成路由
    layoutFilename?: string; // 默认值'layout.vue'
    extensions?: string[]; // 默认值 ['vue']
    mpaConfig?: Record<string, MpaConfig>;
}

export interface MpaConfig {
    alias?: string;
    title?: string;
}

mock #

是否开启本地mock, mock仅在serve阶段生效

  • 类型： boolean
  • 默认： false

vConsole #

是否开启vConsole

  • 类型： boolean | VConsolePluginOptions
  • 默认： false
export interface VConsolePluginOptions {
    enable?: boolean;
    src?: string;
    theme?: 'light' | 'dark';
}

autoImport #

是否开启unplugin-auto-import

  • 类型： boolean | Record<string, any>
  • 默认： true

提示

  • vue2默认导入了vue-demi, vue3默认导入了vue
  • 如果要自定义导入参考unplugin-auto-import文档

analyzer #

是否开启打包分析

  • 类型： boolean
  • 默认： false

inspect #

是否开启vite-plugin-inspect, 仅在serve阶段生效

  • 类型： boolean
  • 默认： false

legacy #

来自@vitejs/plugin-legacy

  • 类型： boolean | string | string[] | Record<string, string> | LegacyPluginOPtions
  • 默认： false
