fetcher #

fetcher是vue-model-lite的请求库, 基于fetch封装, 提供了类axios的api

  • 支持timeout
  • 支持abort
  • credentials默认为same-origin
import { fetcher } from 'vue-model-lite';
fetcher({
    url: 'https://example.com',
});

api #

fetcher(options);
// fetcher也提供了各种method的简写
fetcher.get(url, options?);
fetcher.post(url, options?);
fetcher.put(url, options?);
fetcher.delete(url, options?);
fetcher.patch(url, options?);
fetcher.head(url, options?);
fetcher.options(url, options?);

options #

{
    url: '',
    method: 'get', // 默认为get
    timeout: 0, // 超时时间, 默认为0, 不超时
    base: '', // 如果url不为绝对路径, 会拼在url前面
    data: {}, // get类请求会拼接在url上, post在body传输
    headers: {},
    credentials: 'same-origin', // 默认为same-origin
    requestType: 'json', // json | form, 默认json
    responseType: 'json', // json | text | stream, 默认为json
    transformRequest: [(data) => data], // 处理请求
    transformResponse: [(data) => data], // 处理响应
    validateStatus: (status: number) => boolean, // 自定义response.status
    fetch: fetch, // 自定义fetch实例，默认使用浏览器原生fetch
}

response #

{
    // server返回的data
    data: {}
    // 以下属性同fetch response
    ok: true,
    status: 200,
    statusText: 'OK',
    headers: {},
    type: 'cors',
    url: '',
}

create #

fetcher.create(options)

// 创建一个新的实例
const api = fetcher.create({
    timeout: 1000,
});

// 在Api的配置上创建另一个实例
const newApi = api.create({
    credentials: 'include',
});

config #

fetcher.config(options)

// 修改默认fetcher的timeout为1000
fetcher.config({
    timeout: 1000,
});

// 也可以修改实例的默认配置
const api = fetcher.create({
    timeout: 2000,
});

// api的timeout将修改为3000
api.config({
    timeout: 3000,
    requestType: 'form',
})

form #

fetcher默认已json方式发送post请求, 如果需要已form格式发送请求, 有两种方式：

  1. 修改requestType为form即可
  2. 传输的data设置为FormData格式 大部分场景下都不需要手动处理header['content-type']
