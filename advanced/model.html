Model #

组件与状态分离 #

如果你喜欢组件与状态分离, 可以使用createModel来创建一个model

model可以被任意组件引用, 在组件间共享状态

createModel #

createModel的参数是个函数, 返回一个reactive的model

// profileModel.ts
import { createModel, useFetch } from 'vue-model-lite';
import { computed, ref } from 'vue';
import { photoModel } from './photoModel';

export const profileModel = createModel(() => {
    // 可以引用其他model
    const { getPhoto } = photoModel;

    // 可以使用hooks
    const { data } = useFetch({
        url: 'https://profile'
    });

    // 可以定义计算属性
    const avatar = computed(() => data.value?.avatar ?? '');

    // 用户自定义的变量
    const userState = ref('');

    // 用户自定义的方法
    const func = () => {};

    return {
        avatar,
        getPhoto,
        func,
    }
});


<template>
    <img :src="profileModel.avatar" @click="profileModel.getPhoto">
</template>
<script setup>
import { profileModel } from './profileModel';
import { isReactive } from 'vue';

console.log(isReactive(profileModel)); // true;
</script>

选项式 API #

对选项式API, 可以通过计算属性使用model

<template>
    <img :src="profileModel.avatar" @click="profileModel.getPhoto">
</template>
<script>
import { profileModel } from './profileModel';

export default {
	computed: {
        profileModel() {
            return profileModel;
        }
    }
</script>

共享状态 #

model独立于组件之外，vue-model-lite内部会创建一个Map来缓存model,

所以你在任意多组件之中引用同一个model, 返回的都是同一份数据

提示

  • 如果model里useFetch是auto, 在import时就会发起请求

DevTools支持 #

createModel只是语法糖, 最终返回的都是reactive对象, 可以原生被Vue Devtools支持
