多页应用 #

Vite原生支持多页应用, Veno改进了他

Veno默认是单页应用, 多页应用需要使用配置文件指定

// veno.config.js
export default defineConfig({
    router: {
        mpa: true, // 使用mpa模式
        filename: 'main.ts' // 指定使用main.ts生成路由
    }
})

多页应用的目录结构 #

src/pages/
  ├── user/
  │  ├── main.ts
  ├── activity/
  │  ├── activity-a
  │  │  ├── main.ts
  │  │  ├── index.html
  │  ├── activity-b
  │  │  ├── main.ts

如上目录结构将会生成三条路由

  • /user
  • /activity/activity-a
  • /activity/activity-b

提示

  • 路由可以在浏览器直接访问
  • 可以使用base给路由增加前缀

模板复用与自定义 #

多页应用的模板会复用根目录下的index.html

如果某些路由需要自定义html内容, 可以在对应目录下建一个index.html

注意

  • index.html不需要script引入入口文件, Veno会自动处理

更多配置 #

可以通过mpaConfig对路由进行更精细化的控制和扩展

// veno.config.js
export default defineConfig({
    router: {
        mpa: true, // 使用mpa模式
        filename: 'main.ts' // 指定使用main.ts生成路由
        mpaConfig: {
            '/user': {
                alias: '/profile', // 访问profile将会渲染/user
                title: '用户主页',
                // ...
            }
        }
    }
})
